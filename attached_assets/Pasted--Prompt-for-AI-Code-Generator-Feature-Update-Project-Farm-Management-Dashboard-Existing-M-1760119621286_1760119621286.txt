## Part 1: Backend Development (Node.js, Express & MySQL)

### Task 1: Integrate MySQL
1.  **Install MySQL Driver:** Add the `mysql2` package to your project's dependencies (`npm install mysql2`).
2.  **Create MySQL Connection:** Create a new configuration file (e.g., `config/mysql.js`) to handle the connection to your MySQL database.
3.  **Update Environment Variables:** Add your MySQL connection details (HOST, USER, PASSWORD, DATABASE) to the `.env` file.

### Task 2: Create the MySQL `orders` Table
Provide the following SQL schema for a new table named `orders`. This will store all the order information.

```sql
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id VARCHAR(255) NOT NULL UNIQUE,
    order_details TEXT NOT NULL,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    user_mongo_id VARCHAR(255) NOT NULL
);
order_id: A unique ID that will be automatically generated (e.g., using a library like uuid or a custom function) when an order is created.

order_details: The simple string/text that the user enters for their order.

user_mongo_id: This field will store the user's _id from the MongoDB users collection to link the order to a specific user.

Task 3: Create New API Endpoints for Orders
Create a new route file (e.g., routes/orders.js) for handling the following actions. These routes must be protected by your existing JWT authentication middleware.

POST /api/orders/create

This endpoint will create a new order.

It should accept orderDetails in the request body.

Logic:

Get the user's MongoDB _id from the decoded JWT token.

Generate a unique order_id (e.g., ORD- followed by a timestamp or random characters).

Insert a new row into the MySQL orders table with the new order_id, the user's orderDetails, and their user_mongo_id.

Return a success message.

GET /api/orders

This endpoint will fetch all orders for the currently logged-in user.

Logic:

Get the user's MongoDB _id from the decoded JWT token.

Query the MySQL orders table to find all records where user_mongo_id matches the user's ID.

Return the list of orders as a JSON array.
